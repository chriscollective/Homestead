# 祖傳家園規劃系統：規格與目標計劃書

版本：v0.1（草案）
日期：2026-07-19
專案代號：Homestead Planner（暫定，中文名待定）

---

## 1. 專案概述

### 1.1 願景

打造一個以《鳴響雪松》祖傳家園理念為核心的**土地規劃工具**，讓使用者能在數位環境中設計自己的一公頃家園：繪製真實的不規則地界、配置森林／果樹／菜園／水體／房屋，並透過時間軸模擬看見家園在 1 年、5 年、20 年、50 年後的樣貌。

核心信念：祖傳家園是為世代而設計的，規劃工具必須能呈現「時間」這個維度。

### 1.2 定位

- **是**：規劃工具（實用導向），幫助真正想建立家園的人做土地設計決策
- **不是**：模擬經營遊戲、專業 GIS 軟體、建築 CAD

### 1.3 差異化

1. **台灣在地化**：台灣氣候分區、本土樹種果樹資料庫、不規則地形與坡地地勢的第一級支援
2. **時間軸模擬**：不只畫平面圖，而是模擬家園隨年份的成長演變
3. **祖傳家園設計原則內建**：森林佔比、綠籬圍合、水源配置等原則作為即時檢查規則
4. **2D + 3D 雙模式**：平面規劃效率 + 立體空間感受

### 1.4 目標使用者

| 使用者 | 需求 |
|---|---|
| 鳴響雪松讀者／家園夢想者 | 把心中的家園具象化，反覆試驗配置 |
| 已有土地的實踐者 | 匯入真實地界與地勢，做實際種植規劃 |
| 生態村／聚落發起人 | 多家園組合的聚落層規劃（Phase 4） |
| 樸門（Permaculture）愛好者 | 分區設計、水線規劃、微氣候分析 |

### M1 土地畫布（2D）

**邊界系統**
- 手繪多邊形邊界（點擊逐點繪製、拖曳修改頂點、插入／刪除頂點）
- 支援凹多邊形、任意不規則形狀（台灣地籍現實）
- 即時顯示：面積（㎡／公頃／坪）、周長
- 比例尺設定：使用者輸入實際尺寸或依地籍資料校正
- 進階（P3+）：匯入地籍圖／GeoJSON／KML；底圖描圖（上傳空照圖或地籍謄本影像描邊）

**畫布操作**
- 縮放（滾輪／手勢）、平移、網格顯示切換（1m／5m／10m 網格）
- 指北針與方位設定（影響光照與風向分析）
- 圖層系統：地勢層、水體層、植栽層、建物層、分析層，可個別顯示／隱藏
- 復原／重做（undo/redo stack）
- 測距工具、面積量測工具

### M2 元素系統

**元素類別**
1. **喬木**：果樹（單株）、林木（單株或成片林地區塊）
2. **灌木／綠籬**：沿線繪製，自動計算所需株數
3. **菜園區**：多邊形區塊，內部可指定輪作作物
4. **水體**：池塘（多邊形）、水井（點）、溪流（線）、雨水收集
5. **建物**：住宅、工具間、雞舍、蜂箱、溫室、涼亭
6. **路徑**：步道、車道
7. **其他**：堆肥區、火堆區、草地／野地

**放置與編輯**
- 拖放放置、旋轉、複製、群組
- 單株植物顯示樹冠投影圈（依當前年份的冠幅）
- 成片區塊：指定密度自動散布（如每 4m 一株的混合林）
- 碰撞／間距提示：兩株喬木過近時警示（依成熟冠幅）
- 元素屬性面板：物種、種植年份、備註

**設計原則檢查（祖傳家園規則引擎）**
- 森林覆蓋比例（目標 ~50%）即時儀表
- 綠籬圍合度檢查（邊界被綠籬覆蓋的百分比）
- 水源存在性與可及性
- 可自訂規則開關（不強制，僅提示）

### M3 植物資料庫

**資料欄位（PlantSpecies）**
- 基本：中文名、學名、別名、類別（喬木／灌木／草本／蔓藤）
- 成長：成熟年限、各年份高度曲線、各年份冠幅曲線、根系範圍、壽命
- 環境需求：日照（全日照／半日照／耐陰）、需水量、適合坡度、耐風性、土壤偏好
- 氣候：適生氣候分區（台灣北／中／南／東／山地）、耐寒／耐熱區間
- 產出：結果始年、年產量估計（kg）、產季月份、熱量密度（供 M9）
- 功能標籤：防風、蜜源、固氮、誘鳥、藥用、建材、薪柴
- 食物森林層次（forestLayer）：樹冠／林下／灌木／草本／地被／爬藤／根系（供 M13）
- 台灣特別欄位：是否原生種、颱風耐受性

**初期收錄範圍（P1 目標 30 種）**
- 果樹：芒果、龍眼、荔枝、柑橘類、芭樂、木瓜、香蕉、酪梨、蓮霧、柿
- 林木：樟樹、台灣肖楠、烏心石、光蠟樹、楓香、青剛櫟、相思樹
- 綠籬／灌木：七里香、朱槿、金露花、山茶
- 竹類：綠竹、孟宗竹
- 蔬菜以「菜園區塊 + 輪作組合」抽象處理，不逐種建模

**資料管理**
- 後台 CRUD（自用管理介面即可）
- 資料來源標註欄位（利於日後查證修正）
- 使用者自訂物種（P3）

### M4 時間軸模擬

- 年份滑桿：第 0～50 年（可延伸至 100 年）
- 所有植物依種植年份 + 成長曲線插值呈現當年冠幅／高度
- 關鍵年份快照對比（並排顯示第 1 vs 第 10 年）
- 事件標記：某年新種、某年砍伐／移除（元素有生命週期）
- 季節切換（P3，配合 M7）：春夏秋冬的落葉、產季顯示

### M5 地勢系統

- 高度圖（heightmap）資料結構：邊界範圍內的規則網格高程（建議 1~2m 解析度）
- 高程編輯：筆刷抬升／下降、平滑、設定固定高程點後內插
- 等高線自動生成與顯示（可調間距 0.5m／1m）
- 坡度熱圖（分析層）：標示 >15°、>30° 區域
- 剖面工具：畫一條線，顯示沿線地形剖面圖
- 進階（P4）：匯入政府 DEM（20m／1m 數值地形模型）自動生成地勢

### M6 3D 檢視

- 與 2D 共用同一份資料模型，僅為另一個渲染器（單一資料源原則）
- 地形：heightmap 生成起伏網格 + 材質（草地／土／水）
- 植物：低模（low-poly）樹模型，依物種類別 + 當前年份高度冠幅縮放；遠景用 billboard／instancing 優化
- 建物：預設房型的簡易 3D 模型
- 相機：軌道環繞、第一人稱漫遊模式（走在自己的家園裡）
- 光影：即時太陽光影（與 M7 光照分析共用太陽位置計算）
- 效能目標：一公頃、500+ 株植物在中階裝置維持 30fps（instancing 必要）

### M7 環境分析

**光照**
- 太陽軌跡精確計算（緯度 + 日期 + 時刻），台灣預設 23.5°N 附近可依地點調整
- 2D 陰影分析層：指定日期時刻，顯示樹冠與建物投影
- 全日／全年日照累積熱圖（哪些區域是全日照、哪些長期遮蔭）
- 冬至／夏至極端情境快速切換

**水流**
- 由 heightmap 計算坡度梯度，模擬地表逕流方向（D8 演算法等級即可）
- 集水區與匯流線視覺化 → 建議池塘／swale（等高集水溝）位置
- 池塘集水量粗估（集水面積 × 年雨量 × 逕流係數）

**風向**
- 台灣季風模型：冬季東北季風、夏季西南風、颱風情境（方向性參數）
- 防風林效果視覺化：綠籬／林帶背風側的減風區域（簡化規則：樹高 × 倍數）
- 建議防風林配置提示

**季節**
- 產季月曆：全年每月有哪些作物產出（甘特圖式視覺）
- 落葉樹冬季透光的差異納入光照分析（P4）

### M8 房屋模組

- **預設房型庫**起步（刻意不做自由建模，避免變成迷你 SketchUp）：小木屋、平房、二層宅、土團屋等 5~8 種
- 可調參數：尺寸縮放、朝向旋轉、門窗方位標記
- 朝向分析：結合 M7 光照，顯示主要開窗面的日照時數
- 房屋周邊建議：遮蔭樹位置、防風配置提示
- 遠期（P4+）：簡易隔間編輯器、與外部工具（如 Sweet Home 3D 格式）的匯入匯出評估

### M9 自給自足計算

- 輸入：家庭人數、飲食型態（預設含蛋奶素選項）
- 產出估算：全年蔬果產量（依 M3 資料 × 當前年份）、熱量供給比例
- 柴火估算：林地面積與樹種的年可持續採伐量 vs 家庭需求
- 水資源：雨水收集 + 池塘容量 vs 用水需求
- 「自給率儀表板」：食物／能源／水三項達成率，隨時間軸年份變動（第 3 年 30% → 第 15 年 90% 的成長曲線）

### M12 能源模組

家園可能與河水相鄰、或位於強風地帶（如新竹），能源配置應依基地條件因地制宜。本模組與 M7 環境分析深度整合：**光照分析餵太陽能、水流分析餵微水力、季風模型餵風力**。

**太陽能（Solar PV）**
- 元素：屋頂型光電板（附著於建物）、地面型光電陣列（多邊形區塊）、太陽能熱水器
- 發電估算：面板面積 × 效率 × 該位置年日照量（來自 M7 日照累積熱圖，自動扣除樹蔭遮蔽損失）
- 遮蔽警示：時間軸快轉時，若未來某年樹冠長大遮到面板，主動提示（這是本工具獨有的價值——一般太陽能評估工具不會算「20 年後那棵樟樹」）
- 傾角與朝向建議：依緯度給出最佳傾角，朝向偏差的效率折損顯示

**微水力（Micro-hydro）**
- 適用條件：家園與溪流相鄰或有溪流穿越（M2 溪流元素）
- 關鍵參數：流量（L/s）與落差（head，由 M5 地勢沿溪流線自動計算高程差）
- 元素：取水點、壓力管路線、水輪機位置、回水點
- 發電估算：P = ρ × g × Q × H × η（簡化公式，η 預設 0.5~0.7）
- 提示事項：枯水期流量折減參數、水權法規提醒（台灣引水須申請水權，UI 標註法規注意事項連結）

**風力（小型風機）**
- 適用條件：季風強勁地帶（新竹、恆春、澎湖、沿海）
- 元素：小型風機（點位 + 塔高參數）
- 風況輸入：依地點選擇風區等級（可先做粗分級：強風區／一般／弱風區），結合 M7 季風方向
- 配置檢查：風機與防風林的矛盾提示——防風林保護作物但會削弱風機，兩者位置需錯開（風機置於盛行風上風處或防風林減風區之外）
- 發電估算：簡化功率曲線 × 年等效滿載小時數（依風區等級查表）
- 噪音與距離提示：與住宅建議距離

**能源儀表板（併入 M9 自給自足儀表板）**
- 家庭用電需求估算（人數 + 生活型態）
- 三種能源的年發電量合計 vs 需求 → 能源自給率
- 季節互補視覺化：夏季太陽能強、冬季東北季風風力強、雨季水力強——顯示逐月供需曲線，凸顯互補配置的價值
- 儲能粗估：依供需落差建議電池容量級距（僅級距建議，不做精細電力工程）

**明確簡化（納入 Non-goals 精神）**
- 不做電路設計、逆變器選型、併網申請流程
- 所有估算標示為「規劃參考值」，實際安裝需專業評估

### M13 樸門設計工具（Permaculture）

樸門（Permaculture）是與祖傳家園高度互補的設計體系——祖傳家園給「願景與精神」，樸門給「方法論」。三大倫理：照顧地球、照顧人、分享多餘。以下是最適合工具化的四個概念：

**分區分析（Zones 0-5）**
- 樸門依「使用頻率」將土地分為 0~5 區：Zone 0 住家、Zone 1 每日往返的生活區（廚房菜園、育苗）、Zone 2 食物森林與小型家禽、Zone 3 主要作物區、Zone 4 半野放林地（薪柴、採集）、Zone 5 完全野地（留給自然）
- 工具化：以住家為中心的**距離環疊圖層**（分析層），自動或手動劃定各區
- **頻率-距離檢查**：需要每天照顧的元素（菜園、雞舍）若放在離家太遠處，給出提示——「這個菜園距住家 80m，每日往返成本高，建議移入 Zone 1」
- 這與祖傳家園的空間觀（森林在外圍、生活在核心）天然吻合

**扇形分析（Sector Analysis）**
- 分析從基地「外部」進入的能量與影響：夏季日照扇形、冬季低角度日照、東北季風扇形、西南季風、颱風主要路徑方向、噪音源（道路）、景觀視野、水患方向
- 工具化：以基地為中心的**扇形圖疊層**，每個扇形可標註類型與角度範圍
- M7 的光照與風向分析本質上就是自動化的扇形分析——M13 把它提升為正式的設計圖層，並加入日照風向以外的人文扇形（噪音、視野、鄰地影響）
- 設計建議連動：防風林建議自動對齊季風扇形、觀景窗朝向視野扇形

**食物森林層次（Food Forest Layers）**
- 七層結構：樹冠層（芒果、龍眼、波羅蜜）、林下層（咖啡、酪梨、香蕉）、灌木層（桑椹、金棗、洛神）、草本層（薑科、鳳梨、月桃）、地被層（薄荷、地瓜葉、佛手瓜）、爬藤層（山藥、愛玉、川七）、根系層
- 工具化：M2 的區塊元素新增「食物森林」類型，內部以**層次配方**定義（每層選物種 + 密度），而非逐株放置
- **層次完整度檢查**：分析一個食物森林區塊缺哪些層（「此區塊缺地被層，裸露土壤易生雜草與水分流失」）
- M3 資料庫增加欄位：`forestLayer`（該物種所屬層次）
- 3D 呈現時食物森林的垂直層次感會非常直觀——這是 3D 模式的殺手級展示場景

**元素關係分析（Relative Location）**
- 樸門核心原則：每個元素供多種功能、每個功能由多個元素支持，元素間的相對位置創造互利關係
- 工具化（輕量版）：元素關係規則表，例如——雞舍鄰近菜園（雞糞堆肥、雞吃菜渣）、蜂箱鄰近果樹（授粉）、香蕉圈鄰近廚房出水口（污水處理）、池塘位於菜園上坡（重力給水）
- 放置元素時顯示**協同提示**：「附近 15m 內有果樹，蜂箱放這裡能提升授粉效益 ✓」
- 不做強制，只做正向提示（符合工具「建議不強制」的一貫哲學）

**新增樸門特色元素（併入 M2 元素清單）**
- 等高集水溝（swale）：沿等高線的線元素，與 M5 地勢連動自動吸附等高線
- 香蕉圈（banana circle）：點元素，污水處理 + 食物生產
- 鎖眼菜園（keyhole garden）、螺旋香草園（herb spiral）：特色菜園形狀模板
- 移動雞舍（chicken tractor）：可標註輪替路徑的特殊元素
- 火箭爐／麵包窯／太陽能鍋：Zone 0 周邊的烹煮設施元素

**設計原則儀表擴充**
- 原有的祖傳家園規則（森林 50%、綠籬圍合）之外，可切換樸門檢查組：分區合理性、Zone 5 野地保留比例、食物森林層次完整度、元素協同數

### M10 聚落模式

- 多個家園單元（每戶可載入既有設計）拼接於一張聚落地圖
- 共享設施元素：學校、集會所、共同倉儲、道路網、共用水源
- 聚落層檢查：道路可及性、公共空間比例
- 協作（遠期）：多使用者共同編輯、家園間相鄰協調（例如共用綠籬）

### M11 帳號、儲存與分享

- 註冊登入（JWT，含 refresh token——順便補上你想練的技能點）
- 專案 CRUD：多個家園設計、版本快照（設計的「存檔點」）
- 自動儲存與雲端同步
- 匯出：PNG 平面圖、PDF 規劃書（含元素清單、物種列表、自給率報告）、GeoJSON
- 分享：唯讀連結分享、社群展示頁（遠期，類似作品藝廊）

## 3. 技術規格

### 3.1 技術棧

| 層 | 選型 | 理由 |
|---|---|---|
| 前端框架 | React 18 + TypeScript + Vite | 既有技術棧 |
| 2D 渲染 | SVG（React 元件）起步；元素量大時評估 Canvas/Konva | SVG 開發快、事件處理自然，一公頃規模通常足夠 |
| 幾何運算 | Turf.js | 多邊形面積、point-in-polygon、buffer、切割 |
| 3D 渲染 | Three.js + react-three-fiber + drei | React 生態整合最佳 |
| 狀態管理 | Zustand | 輕量、適合畫布類高頻更新；undo/redo 用 patch 序列 |
| 後端 | Node.js + Express + TypeScript | 既有技術棧 |
| 資料庫 | MongoDB（Mongoose） | 設計檔為巢狀文件，天然契合；GeoJSON 有 2dsphere 支援 |
| 驗證 | JWT + refresh token rotation | 技能目標 + 實際需求 |
| 部署 | 前端 Vercel／後端 Railway 或 VPS + Docker | Docker 為技能目標 |
| 測試 | Vitest（幾何與模擬邏輯必測）+ Playwright（關鍵流程） | 演算法正確性優先 |

### 3.2 核心資料模型（草案）

```typescript
// 家園設計檔
interface HomesteadProject {
  _id: string;
  ownerId: string;
  name: string;
  location: { lat: number; lng: number; climateZone: ClimateZone };
  boundary: GeoJSON.Polygon;        // 公尺座標系（本地平面座標）
  terrain: {
    resolution: number;              // 網格解析度（m）
    grid: number[][];                // 高程值
  };
  elements: PlacedElement[];
  settings: { northAngle: number; gridVisible: boolean; /* ... */ };
  snapshots: VersionSnapshot[];
  createdAt: Date;
  updatedAt: Date;
}

// 放置元素（discriminated union）
type PlacedElement =
  | PlantElement      // { kind:'plant'; speciesId; position; plantedYear; removedYear? }
  | AreaElement       // { kind:'area'; areaType:'forest'|'garden'|'meadow'; polygon; speciesMix? }
  | WaterElement      // { kind:'water'; waterType:'pond'|'well'|'stream'; geometry }
  | BuildingElement   // { kind:'building'; modelId; position; rotation; scale }
  | PathElement       // { kind:'path'; pathType; line; width }
  | HedgeElement      // { kind:'hedge'; speciesId; line; spacing }
  | EnergyElement;    // 見下方

// 能源元素
type EnergyElement =
  | { kind:'energy'; energyType:'solar_roof';   buildingId: string; areaM2: number; tiltDeg: number; azimuthDeg: number }
  | { kind:'energy'; energyType:'solar_ground'; polygon: GeoJSON.Polygon; tiltDeg: number; azimuthDeg: number }
  | { kind:'energy'; energyType:'micro_hydro';  intakePoint: Point; turbinePoint: Point; pipeline: LineString; flowLps: number }
  | { kind:'energy'; energyType:'wind_turbine'; position: Point; towerHeightM: number; ratedKw: number };

// 植物物種
interface PlantSpecies {
  _id: string;
  nameZh: string;
  nameSci: string;
  category: 'tree_fruit' | 'tree_forest' | 'shrub' | 'bamboo' | 'vine';
  growth: {
    matureYears: number;
    heightCurve: YearValue[];       // [{year, value}] 插值用
    canopyCurve: YearValue[];
    lifespan: number;
  };
  needs: {
    sun: 'full' | 'partial' | 'shade';
    water: 1 | 2 | 3;
    slopeRange: [number, number];
    windTolerance: 1 | 2 | 3;       // 3 = 耐颱風
  };
  climateZones: ClimateZone[];
  yield?: {
    startYear: number;
    kgPerYearCurve: YearValue[];
    harvestMonths: number[];
    kcalPerKg: number;
  };
  tags: PlantTag[];                  // 'windbreak' | 'nectar' | 'nitrogen' | ...
  isNative: boolean;
  sources: string[];
}
```

### 3.3 架構原則

1. **單一資料源、雙渲染器**：2D 與 3D 都是 `HomesteadProject` 的純函式視圖，任何模擬邏輯不得寫在渲染層
2. **模擬引擎獨立**：成長插值、光照、水流、產量計算全部是純函式模組（`/engine`），可單元測試、可在 Web Worker 執行
3. **本地優先**：編輯操作即時反映本地狀態，背景同步至後端（防止斷線丟失設計）
4. **座標系統**：內部統一使用公尺為單位的本地平面座標，僅在匯入匯出時轉換經緯度
5. **效能預算**：2D 首次渲染 <1s、拖曳 60fps；3D 中階筆電 30fps；重運算（全年光照熱圖）進 Web Worker 並顯示進度

### 3.4 演算法備忘

- 光照：NOAA 太陽位置演算法（方位角 + 仰角），陰影 = 元素高度 × 幾何投影；累積熱圖 = 逐時取樣加總
- 水流：D8 flow direction + flow accumulation（經典地表水文演算法，網格版易實作）
- 成長：曲線關鍵點線性插值（未來可換 S 曲線）
- 等高線：Marching Squares
- 防風：背風減風區 = 樹高 × 10~15 倍距離的漸衰簡化模型

## 5. 風險與對策

| 風險 | 等級 | 對策 |
|---|---|---|
| 範圍膨脹（最大風險） | 高 | 嚴守 Phase 界線；房屋自由建模、專業 GIS 功能明確列為「不做」 |
| 植物資料正確性 | 中 | 標註資料來源；MVP 標示「估算值」；長期開放社群修正 |
| 3D 效能 | 中 | instancing、LOD、低模美術風格（刻意選擇手繪低模風，兼顧效能與美感） |
| SVG 效能瓶頸 | 低-中 | 元素 >1000 時遷移 Konva/Canvas，資料層不變所以遷移成本可控 |
| 環境模擬過度簡化被質疑 | 低 | 定位為「規劃輔助」非「科學模擬」，UI 明示簡化假設 |
| 個人時間（本業 + LegPrint + 日文） | 高 | 每 Phase 有獨立可交付價值，隨時可暫停而不留半成品 |

## 6. 明確不做（Non-goals）

- 自由建築建模（用預設房型代替）
- 專業測量精度的 GIS 功能
- 遊戲化養成系統（金幣、任務、成就）
- 即時多人（Phase 4 前）
- 原生 App（Web 優先，RWD 支援平板檢視）

## 7. 成功指標

- **P1**：自己用它完成一份滿意的家園設計（dogfooding）
- **P2**：3 位鳴響雪松社群朋友實際使用並回饋
- **P3**：至少 1 位「真的有土地」的使用者用它做實際規劃
- **長期**：成為華文圈祖傳家園／樸門規劃的預設工具；累積可信的台灣物種資料庫
